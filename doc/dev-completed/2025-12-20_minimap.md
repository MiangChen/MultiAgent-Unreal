# 小地图 (Mini-map) 设计方案

> 日期: 2025-12-20
> 状态: 📋 设计中

## 功能需求

1. **位置**: 左上角，圆形
2. **显示内容**:
   - 地图俯视图（建筑轮廓、障碍物）
   - Agent 位置图标（不同类型不同颜色/图标）
   - 相机视野指示器（三角形箭头）
3. **交互**:
   - 点击小地图：相机跳转到该位置
   - 可选：拖动移动相机

## 技术方案

### 方案 A: Scene Capture + UMG Widget（推荐）

使用 `SceneCapture2D` 从上方拍摄场景，渲染到 `RenderTarget`，在 UMG 中显示。

**优点**:
- 自动显示真实地图内容
- 支持动态更新
- 可以用材质做圆形遮罩

**缺点**:
- 性能开销（需要额外渲染）
- Agent 可能太小看不清

**实现步骤**:
1. 创建 `SceneCapture2D` Actor，放在地图上方
2. 创建 `RenderTarget` 纹理
3. 创建 UMG Widget 显示 RenderTarget
4. 在 Widget 上叠加 Agent 图标
5. 处理点击事件

### 方案 B: 纯 Canvas 绘制

在 HUD 中用 Canvas 绘制简化的地图和图标。

**优点**:
- 性能好
- 完全可控

**缺点**:
- 需要手动绘制地图轮廓
- 不能自动反映地图变化

### 方案 C: 混合方案

- 背景：静态地图图片（预先截图或手绘）
- 前景：Canvas 绘制 Agent 图标和相机指示器

**优点**:
- 性能好
- 背景可以美化

## 推荐方案: A (Scene Capture)

### 组件结构

```
MAMiniMapManager (Actor)
├── SceneCapture2D (俯视相机)
├── RenderTarget (渲染目标)
└── 配置参数 (地图范围、缩放等)

MAMiniMapWidget (UMG Widget)
├── 圆形背景 (RenderTarget 材质)
├── Agent 图标层 (动态生成)
├── 相机视野指示器
└── 点击事件处理
```

### 关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| MapSize | 小地图覆盖的世界范围 | 10000 x 10000 |
| WidgetSize | 屏幕上的大小 | 200 x 200 px |
| CaptureHeight | 俯视相机高度 | 5000 |
| UpdateRate | 更新频率 | 10 FPS |

### Agent 图标

| 类型 | 颜色 | 图标 |
|------|------|------|
| Drone | 蓝色 | ▲ 三角形 |
| RobotDog | 绿色 | ● 圆形 |
| Human | 黄色 | ■ 方形 |
| 选中的 | 白色边框 | - |

### 相机指示器

- 三角形箭头，指向相机朝向
- 扇形区域表示视野范围
