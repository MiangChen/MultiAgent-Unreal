# 鼠标模式与视角控制

> 日期: 2025-12-20
> 状态: ✅ 已实现

## 交互设计

| 操作 | 功能 |
|------|------|
| **左键拖动** | 框选 Agent |
| **左键单击 Agent** | 选中该 Agent |
| **左键单击空地** | 清除选择 |
| **右键拖动** | 旋转视角（Yaw/Pitch） |
| **中键** | 导航**已选中的** Agent 到点击位置 |
| **M 键** | Select ↔ Deployment 切换（需有待部署单位） |

## 模式说明

### Select 模式（默认）
- 左键框选/选中 Agent
- 右键拖动旋转视角
- 中键导航已选中的 Agent

### Deployment 模式
- 左键框选区域放置待部署的 Agent
- 右键拖动旋转视角
- 背包有单位时才能进入（M 键切换）

---

## 提案：合并 Select 和 Navigate 模式

### 背景
现在 Select 和 Navigate 模式的功能高度重叠，且 Human 有特殊的左键导航逻辑，增加了复杂性。

### 合并方案（已确认）

**只保留 Select 模式**，统一所有 Agent 的操作逻辑：

| 操作 | 合并后功能 |
|------|-----------|
| **左键拖动** | 框选 Agent |
| **左键单击 Agent** | 选中该 Agent |
| **左键单击空地** | 清除选择 |
| **右键拖动** | 旋转视角 |
| **中键** | 导航**已选中的** Agent（Human/RobotDog/Drone 统一） |
| **M 键** | Select ↔ Deployment 切换 |

### 关键改动
1. 删除 Navigate 模式
2. 删除 Human 的左键特殊导航
3. 中键导航改为只对**已选中的 Agent** 生效
4. 所有 Agent 类型（Human、RobotDog、Drone）使用相同的导航逻辑

### 优点
1. 操作逻辑统一，无特殊情况
2. 必须先选中才能导航，更可控
3. 减少模式数量，更简单

## 实现细节

### 右键视角旋转
- `OnRightClickPressed`: 记录起始位置，开始旋转模式
- `OnRightClickReleased`: 结束旋转
- `Tick`: 计算鼠标移动 Delta，旋转 Pawn

```cpp
// Tick 中的旋转逻辑
if (bIsRightMouseRotating)
{
    FVector2D Delta = CurrentPos - RightMouseStartPosition;
    if (Delta.Size() > 2.f)
    {
        NewRotation.Yaw += Delta.X * CameraRotationSensitivity;
        NewRotation.Pitch = FMath::Clamp(NewRotation.Pitch - Delta.Y * 0.3f, -89.f, 89.f);
        ControlledPawn->SetActorRotation(NewRotation);
    }
}
```

### 中键导航
- `OnMiddleClick`: 导航已选中的 Agent 到点击位置

### 相关变量
- `bIsRightMouseRotating`: 是否正在右键旋转
- `RightMouseStartPosition`: 右键按下时的鼠标位置
- `CameraRotationSensitivity`: 旋转灵敏度 (默认 0.3)
